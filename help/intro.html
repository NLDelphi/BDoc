Wat is BDoc?

BDoc is een programma dat Delphi projecten uitleest en daar documentatie uit genereert. De documentatie kan bijvoorbeeld worden gebruikt om binnen een team duidelijk te maken welke componenten, standaard routines en interfaces er in een project worden gebruikt. Ook als hulpmiddel bij overdracht van een project is de documentatie zeer waardevol. Door het overzicht dat de BDoc documentatie geeft is het mogelijk om op een overzichtelijke manier te bepalen waar wijzigingen aangebracht moeten worden en wat de gevolgen van deze wijzigingen zijn. Denk hierbij aan classes die erven van een gewijzigde classe en projecten die gebruik maken van deze classes.

De documentatie kan als set HTML bestanden worden aangemaakt, bijvoorbeeld voor publicatie op een intranet of internet. Een tweede optie is het aanmaken van een HTML helpbestand waarin de documentatie van één of meerdere projecten in één bestand wordt aangemaakt. Dat bestand kan vervolgens zonder verdere installatie op elke computer worden gelezen. De documentatie die wordt aangemaakt bestaat uit:
  • Projecten: een overzicht van de projecten met de in dat project gebruikte formulieren, frames en datamodules.
  • Classes: een overzicht van de classes die in de verzameling van projecten is gebruikt. De classes worden in een boomstructuur weergegeven vanaf de standaard Delphi classe. Door in te zoomen op een classe is te zien welke properties en methods in de classe (en zijn ancestors) zijn gedefiniëerd en welke interfaces een classe ondersteund
  • Interfaces: een overzicht van de interfaces die in de verzameling van projecten is gebruikt. Ook de interfaces worden in een boomstructuur weergegeven. Door in te zoomen op een interface is te zien hoe de interface is opgebouwd en welke classes de betreffende interface ondersteunen.
  • Procedures: een overzicht van (niet bij classes behorende) procedures.
  • Types: een overzicht van gedefiniëerde types en constantes binnen de projecten
  • Units: een overzicht van units waaruit de projecten bestaan. Door in de units in te zoomen is te zien welke classes, interfaces, types, constantes en procedures in de unit zijn gedefiniëerd en door welke projecten en andere units betreffende unit wordt gebruikt.
